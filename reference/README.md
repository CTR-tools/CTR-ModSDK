## What do these two folders mean?

TLDR:
* `asm` contains tools for processing the EXE of CTR into usable content for tools like splat, [decomp.me](decomp.me) etc.
* `ghidra` contains a set of ghidra output of CTR. Some types are present, but most are not. Variable names are poor, but @niko has left many comments annotating behavior. Used as a reference for the other projects in `src/`
